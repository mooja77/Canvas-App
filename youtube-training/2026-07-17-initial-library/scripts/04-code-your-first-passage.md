# Create a Codebook and Code Your First Passage in QualCanvas

**Outcome:** Create a meaningful code, apply it to a passage and review the developing codebook.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. Code meaning, not just words (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: Name the idea and keep the source visible
- Narration: A useful code captures what a passage means in relation to the research question. It should be specific enough to guide future decisions and broad enough to recognise again.

### 2. Create a code from the toolbar (0:11)

- Visual: captures/15-code-menu.png
- On-screen callout: Use a concise name, definition and colour
- Narration: Open Code to create a code deliberately. Give it a short name, write a definition that distinguishes it from nearby concepts and use colour as a visual aid, not as the meaning itself.

### 3. Select a defensible passage (0:24)

- Visual: captures/13-canvas-workspace.png
- On-screen callout: Read before and after the segment before applying the code
- Narration: Select the smallest passage that still carries the intended meaning. Check the sentences around it, then apply an existing code or create a new one when the concept is genuinely different.

### 4. Review the codebook as it grows (0:37)

- Visual: captures/23-codebook.png
- On-screen callout: Definitions and counts expose overlap and drift
- Narration: Open the codebook regularly. Codes with similar definitions may need merging; codes with very different examples may need splitting. Counts are prompts for review, not proof that a theme is important.

### 5. Use the command palette for repeat work (0:52)

- Visual: captures/22-command-palette.png
- On-screen callout: Press Control or Command K and search for an action
- Narration: When the canvas becomes familiar, use the command palette to reach common actions without hunting through menus. Search by the outcome you want and keep your attention on the material.

### 6. Let connections reveal repetition (1:03)

- Visual: captures/13-canvas-workspace.png
- On-screen callout: Every coded segment keeps a visible path back to its source
- Narration: As coding continues, connections accumulate between sources and concepts. Use that visual structure to notice repetition, contradiction and gaps, then return to the transcript before drawing a conclusion.

### 7. Code, compare, refine (1:19)

- Visual: Original QualCanvas brand frame
- On-screen callout: The codebook is a working analytical instrument
- Narration: Coding is iterative. Apply a code, compare examples and refine the definition. QualCanvas keeps those decisions visible, but the interpretation remains yours.
