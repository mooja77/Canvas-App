# Build Cases and Compare Participants in QualCanvas

**Outcome:** Create analytical cases, assign source material and compare patterns across participants, sites or groups.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. Compare without flattening the context (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: Cases can represent participants, sites or analytical groups
- Narration: Cross-case analysis is useful when your question depends on comparison. In QualCanvas, a case can represent a participant, site, organisation or another unit that your method defines. Decide that unit before building the comparison.

### 2. Create and describe each case (0:17)

- Visual: captures/32-cases.png
- On-screen callout: Tools → Cases
- Narration: Open Tools and choose Cases. Give each case a safe research label and a short description that explains what it represents. Avoid putting identifiable participant details into the display name.

### 3. Assign the relevant source material (0:32)

- Visual: captures/32-cases.png
- On-screen callout: A useful comparison depends on consistent case membership
- Narration: Link the relevant transcripts or imported responses to each case. Check the assignments carefully: a missing or duplicated source can produce a convincing-looking comparison based on the wrong membership.

### 4. Open Cross-Case analysis (0:46)

- Visual: captures/33-cross-case.png
- On-screen callout: Compare code patterns across defined cases
- Narration: Choose Cross-Case from the Tools menu. QualCanvas summarises how codes appear across the cases you created. Treat the view as a route back to evidence, not as an automatic conclusion.

### 5. Use a framework matrix when the question is structured (1:00)

- Visual: captures/18-analyze-menu.png
- On-screen callout: Case by question is another deliberate comparison
- Narration: For a structured case-by-question view, open Analyze and choose Framework Matrix. The matrix supports systematic comparison while preserving the need to inspect the original excerpts and contextual notes.

### 6. Memo the difference and the exception (1:15)

- Visual: captures/16-memo-menu.png
- On-screen callout: Record rival explanations and negative cases
- Narration: Write down why a difference may matter, what context could explain it and which case does not fit the emerging account. A comparison becomes analytically useful when its reasoning is documented.

### 7. Cases organise comparison—researchers interpret it (1:28)

- Visual: Original QualCanvas brand frame
- On-screen callout: Define · assign · compare · inspect · memo
- Narration: Define the unit, assign sources consistently, compare patterns, return to the evidence and memo your reasoning. Continue in the training centre with the QDPX transfer tutorial when a project needs to move between tools.
