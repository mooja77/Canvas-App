# The Complete QualCanvas Workflow | Transcript to Research Report

**Outcome:** See the full sequence from project setup and coding through analysis, memoing, review and export.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. The complete qualitative workflow (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: Set up · import · code · analyse · memo · review · export
- Narration: This walkthrough connects the whole QualCanvas workflow. The exact method belongs to your study, but the practical sequence is consistent: set up carefully, keep evidence visible and document the decisions that turn coding into an argument.

### 2. 1. Define the workspace (0:15)

- Visual: captures/12-new-canvas.png
- On-screen callout: Name the study and choose a starting structure
- Narration: Create a canvas with a name that identifies the study and phase. Begin blank or use a methodology template, then adapt every starting code or question to the actual protocol.

### 3. 2. Add approved source material (0:28)

- Visual: captures/14-transcript-menu.png
- On-screen callout: Import only what the protocol allows
- Narration: Add transcripts or other supported sources using clear, non-identifying titles where appropriate. Verify each import and keep a source inventory before analysis expands.

### 4. 3. Code in context (0:41)

- Visual: captures/13-canvas-workspace.png
- On-screen callout: Select passages, apply definitions and compare examples
- Narration: Read each passage in context, apply a code that answers the research question and compare new examples with earlier ones. Refine the codebook when meanings drift or overlap.

### 5. 4. Explore patterns (0:54)

- Visual: captures/18-analyze-menu.png
- On-screen callout: Use the tool that matches the analytical question
- Narration: Use text, coding or framework tools to locate patterns and exceptions. Treat every output as a route back to the material, not as an automatic finding.

### 6. 5. Memo the interpretation (1:06)

- Visual: captures/16-memo-menu.png
- On-screen callout: Record claims, uncertainty, alternatives and next checks
- Narration: Write memos as the interpretation develops. Link claims to evidence, note alternative explanations and record why a code, case or theme changed.

### 7. 6. Review quality with the method in mind (1:18)

- Visual: captures/25-intercoder.png
- On-screen callout: Agreement metrics and audit views support—not replace—judgement
- Narration: For team coding, inspect agreement and disagreements. For every study, review the codebook, ethics boundaries and analytical trail before describing the process in a methods section.

### 8. 7. Export for writing and review (1:32)

- Visual: captures/20-export-menu.png
- On-screen callout: Choose a readable, portable and appropriately protected format
- Narration: Export the report, codebook, workbook, visual or exchange file that the next step requires. Verify the result, protect the downloaded copy and keep the original project available for trace-back.

### 9. Keep the chain of reasoning intact (1:47)

- Visual: Original QualCanvas brand frame
- On-screen callout: Source → code → pattern → memo → claim
- Narration: The value of the workflow is the visible chain from source to code, pattern, memo and claim. QualCanvas organises that chain; the researcher remains responsible for its meaning.
