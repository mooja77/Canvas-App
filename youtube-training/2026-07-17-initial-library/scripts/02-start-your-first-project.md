# Start Your First QualCanvas Project | A Calm Beginner Walkthrough

**Outcome:** Create a first canvas, choose a sensible starting structure and know where to get help.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. Your first project, calmly (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: One canvas · one question · one source
- Narration: You do not need to configure every feature before starting. A useful first QualCanvas project needs a clear study name, one research question and one synthetic or properly approved source to practise with.

### 2. Choose the level you actually need (0:13)

- Visual: captures/03-pricing.png
- On-screen callout: Start free; move up only when the study requires it
- Narration: You can begin on the free plan without a card. Check the current plan limits before importing a large study, and use an institutional route when governance or team controls are part of the protocol.

### 3. Your canvases are separate workspaces (0:27)

- Visual: captures/11-canvas-list.png
- On-screen callout: Name each canvas so the study and stage are obvious
- Narration: The canvas list is your project home. Use a specific name that another researcher will recognise, especially when you expect several phases, samples or analytic approaches.

### 4. Pick a useful starting point (0:40)

- Visual: captures/12-new-canvas.png
- On-screen callout: Create a blank canvas or begin from a methodology template
- Narration: Create a blank canvas when the structure should emerge from the material. Use a template when you want a starting codebook or workflow, then edit it to match the study rather than treating it as a fixed method.

### 5. Orient yourself before adding data (0:53)

- Visual: captures/13-canvas-workspace.png
- On-screen callout: Navigator on the left · canvas in the centre · tools across the top
- Narration: The left navigator tracks codes and sources. The centre is the spatial canvas. Primary actions sit across the top, while the status bar shows connection, save state, zoom and the command shortcut.

### 6. Help is available inside the workspace (1:09)

- Visual: captures/27-help-menu.png
- On-screen callout: Product tour, shortcuts and the full guide
- Narration: If the interface feels dense, open Help. Take the product tour, view the keyboard shortcuts or open the complete guide. Learn one outcome at a time instead of trying to memorise every menu.

### 7. A good first session is small (1:24)

- Visual: Original QualCanvas brand frame
- On-screen callout: Create · add one source · make one code · write one memo
- Narration: A successful first session ends with one canvas, one source, one code and one short memo about what you noticed. That is enough to understand the core loop and return with confidence.
