# Find Patterns with QualCanvas Analysis Tools | Queries, Clusters & Frameworks

**Outcome:** Choose the right analysis tool for a question and interpret outputs without treating them as automatic findings.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. Ask a question before choosing a tool (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: An analysis view is evidence to inspect—not a conclusion
- Narration: QualCanvas offers several analysis views, but the tool should follow the question. Decide whether you need to inspect language, coding patterns, cases or a developing framework before opening a chart.

### 2. Analysis is grouped by purpose (0:14)

- Visual: captures/18-analyze-menu.png
- On-screen callout: Text analysis · coding analysis · frameworks and comparison
- Narration: The Analyze menu separates text exploration from coding analysis and comparative frameworks. That structure helps you choose a method instead of clicking every output and keeping only the one that looks persuasive.

### 3. Use the dashboard for orientation (0:28)

- Visual: captures/26-dashboard.png
- On-screen callout: Review progress and distribution before making claims
- Narration: The dashboard is useful for checking coverage, code distribution and project progress. Uneven counts may reveal a sampling issue, a code that is too broad or simply a concept that appears less often.

### 4. Inspect relationships between codes (0:43)

- Visual: captures/18-analyze-menu.png
- On-screen callout: Co-occurrence and Boolean queries answer different questions
- Narration: Co-occurrence shows codes that overlap in passages. Coding queries let you combine conditions with and, or and not. Neither establishes causation; both help you find material worth rereading.

### 5. Bring outputs back to the canvas (0:59)

- Visual: captures/13-canvas-workspace.png
- On-screen callout: Keep the result beside transcripts, codes and memos
- Narration: Analysis nodes live on the same canvas as the source material. Position a result near the relevant codes, then write a memo that records what you think it means and what could contradict it.

### 6. Check the categories behind the output (1:11)

- Visual: captures/23-codebook.png
- On-screen callout: A chart is only as coherent as the codebook beneath it
- Narration: Before reporting a frequency or matrix, inspect the code definitions and examples. If coders used a label inconsistently, refine or recode before treating the result as stable.

### 7. Use tools to return to the data (1:25)

- Visual: Original QualCanvas brand frame
- On-screen callout: Question · inspect · reread · memo · challenge
- Narration: The strongest analysis loop is question, inspect, reread, memo and challenge. Use the visual outputs to locate evidence, then make the argument in your own analytical voice.
