# Import Open-Text Survey Data into QualCanvas

**Outcome:** Import a CSV of open-text responses, map useful columns and prepare responses for coding without exposing respondent data.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. Bring open-text responses onto the canvas (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: A CSV becomes a set of traceable qualitative sources
- Narration: If your survey includes open-text answers, QualCanvas can bring them into the same visual coding workflow as interview material. Start with a clean CSV exported from your survey system, and remove fields you do not need for analysis.

### 2. Open the Survey importer (0:16)

- Visual: captures/31-survey-import.png
- On-screen callout: Survey → Import Survey Data, C S V
- Narration: Inside a canvas, choose Survey to open the CSV importer. Select the file only after checking its first row contains clear column headings and each response occupies a consistent row.

### 3. Map the columns deliberately (0:30)

- Visual: captures/31-survey-import.png
- On-screen callout: Choose a title, response content and optional case field
- Narration: After the file is read, map the column that names each item, the column containing the open-text response and, when appropriate, a case field. Use a research identifier rather than a participant's real name.

### 4. Check the imported sources (0:45)

- Visual: captures/30-sources-panel.png
- On-screen callout: Confirm counts, titles and readable response text
- Narration: Open Sources and spot-check several imported items. Confirm the response count, make sure headings have not become content and check that line breaks and characters survived the CSV export.

### 5. Code responses with the same audit trail (0:59)

- Visual: captures/13-canvas-workspace.png
- On-screen callout: Excerpt → code → pattern → memo
- Narration: Code meaningful excerpts rather than treating every row as a single idea. The source, selected passage, code and later analytical structure remain connected, which makes survey interpretations easier to review.

### 6. Minimise respondent data before import (1:15)

- Visual: captures/24-ethics.png
- On-screen callout: Consent, identifiers and retention rules still apply
- Narration: A CSV can quietly contain email addresses, IP fields or demographic combinations that identify someone. Remove unnecessary columns, follow your consent and retention rules, and use the Ethics panel as a project reminder rather than a substitute for approval.

### 7. Import cleanly, then analyse visibly (1:34)

- Visual: Original QualCanvas brand frame
- On-screen callout: Prepare · map · verify · code · protect
- Narration: The reliable path is simple: prepare a minimal CSV, map the columns, verify the imported sources and then code visibly. Open the QualCanvas training centre for the next tutorial on cases and cross-case comparison.
