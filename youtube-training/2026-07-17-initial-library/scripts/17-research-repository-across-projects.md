# Build a Research Repository Across QualCanvas Projects

**Outcome:** Create a research repository and organise carefully written insights across projects while preserving links back to the source analysis.

**Audience:** Not made for kids  
**Language:** English (Ireland)  
**Voice:** en-IE-EmilyNeural

## Scene plan

### 1. Carry insight across projects without losing its origin (0:00)

- Visual: Original QualCanvas brand frame
- On-screen callout: A repository is a curated layer above individual canvases
- Narration: When research spans several projects, useful insights can disappear into separate files. The QualCanvas Research Repository provides a place to curate insight records across projects while the detailed analysis remains in each canvas.

### 2. Open the plan-controlled Repository (0:16)

- Visual: captures/36-research-repository.png
- On-screen callout: Collect and organise insights across research projects
- Narration: Open Research Repository from your authenticated workspace. Repository access depends on the current QualCanvas plan, so confirm availability before designing a team process around it.

### 3. Name the repository around a durable question (0:29)

- Visual: captures/37-new-repository.png
- On-screen callout: Use a clear name and scope description
- Narration: Choose New Repository and add a name and description. A durable customer journey, service area or research programme usually makes a better boundary than a temporary presentation deadline.

### 4. Keep project analysis in its canvas (0:43)

- Visual: captures/11-canvas-list.png
- On-screen callout: The repository complements canvases; it does not replace them
- Narration: Each canvas remains the home of transcripts, coding and project-level reasoning. Promote a concise insight only after reviewing its supporting evidence, context and limitations.

### 5. Write an insight that can stand up to reuse (0:56)

- Visual: captures/16-memo-menu.png
- On-screen callout: Claim, evidence, context, confidence and limits
- Narration: Before adding an insight, capture the claim, the evidence behind it, the context in which it applies and any uncertainty or negative cases. A memorable sentence without those boundaries can become misleading when reused.

### 6. Agree ownership and review rules (1:11)

- Visual: captures/05-for-teams.png
- On-screen callout: Decide who can add, revise, retire and export insights
- Narration: For a team repository, assign owners and review dates. Retire or revise an insight when later research changes the evidence, and maintain the same participant privacy controls used in the source project.

### 7. Curate insight, not just findings (1:27)

- Visual: Original QualCanvas brand frame
- On-screen callout: Scope · evidence · context · ownership · review
- Narration: A useful repository has a clear scope, traceable evidence, contextual limits, named ownership and regular review. Visit the QualCanvas training centre for applied-research workflows that connect interviews, surveys and team handoffs.
