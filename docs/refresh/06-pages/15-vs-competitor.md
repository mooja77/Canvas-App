# 6.14 `/vs/[competitor]` — comparison pages

← [Pages index](README.md) · [Plan index](../README.md)

**Intent.** Capture comparison search intent. _"NVivo alternative"_ and _"ATLAS.ti alternative"_ are documented buyer-mode keywords. The competitors won't publish honest comparisons themselves; we will, and the honesty itself converts.

## Pages at launch

- `/vs/nvivo`
- `/vs/atlas-ti`
- `/vs/dedoose`

## Section order (every comparison page identical for SEO consistency)

1. **Quick verdict** — 3 short paragraphs. Where each tool fits.
2. **Pricing comparison** — `<CompetitorRow>` showing actual published numbers for both. Source link + date.
3. **Where [competitor] is better** — honest. Examples:
   - NVivo: "Most cited in the literature; if your committee has read 30 papers using NVivo, that matters."
   - ATLAS.ti: "Stronger logo wall (MIT, Boeing, NASA) for buyer credibility."
   - Dedoose: "Active-month billing is genuinely flexible if you code in bursts."
4. **Where QualCanvas is better** — also honest, sourced. Examples: transparent pricing, public changelog, working `/trust`, interactive demo, native dark mode, QDPX export both ways.
5. **Migration path** — short practical section. "Export from NVivo as QDPX; import into QualCanvas." Screenshots.
6. **FAQ** — 5 questions specific to the comparison.

## Voice rule

Never trash the competitor. Every claim has a source. The honesty _is_ the marketing.

## Length target

1,200–1,500 words per page. ~5 min read.

## Caveat

Each page must be re-reviewed quarterly because competitor pricing and features drift.
